# 테스트넷 체크리스트

메인넷 배포 전에 테스트넷에서 다음 항목들을 확인하세요.

## ✅ 필수 체크리스트

### 1. 환경 설정
- [ ] `.env.local`에 `NEXT_PUBLIC_NETWORK=testnet` 설정
- [ ] Base Sepolia 네트워크 RPC URL 설정
- [ ] 테스트넷 토큰 컨트랙트 주소 설정

### 2. 토큰 배포
- [ ] BLA 테스트 토큰 배포 (또는 MockERC20 사용)
- [ ] HUNT 테스트 토큰 배포 (선택사항)
- [ ] USDC 테스트 토큰 확인 또는 배포
- [ ] 각 토큰의 주소를 환경 변수에 추가

### 3. 테스트넷 자금
- [ ] Base Sepolia 테스트 ETH 받기 (Faucet)
- [ ] 각 테스트 토큰을 테스트 지갑에 받기
- [ ] Payment Receiver 주소에 테스트 토큰 전송

### 4. 기능 테스트
- [ ] 지갑 연결 (MetaMask)
- [ ] 네트워크 자동 전환 확인
- [ ] 토큰 잔액 표시 확인

### 5. 결제 테스트
- [ ] BLA 토큰으로 결제
- [ ] HUNT 토큰으로 결제
- [ ] USDC 토큰으로 결제
- [ ] 트랜잭션 승인 및 확인
- [ ] 결제 실패 케이스 테스트 (잔액 부족)

### 6. 음성 메시지 테스트
- [ ] 음성 녹음 기능
- [ ] 음성 업로드 (Supabase Storage)
- [ ] 음성 재생 기능
- [ ] 익명/공개 포스팅 옵션

### 7. 반응 시스템 테스트
- [ ] 이모지 반응 추가
- [ ] 반응 제거
- [ ] BB Points 적립 확인
- [ ] 반응 카운트 표시

### 8. 클레임 시스템 테스트
- [ ] BB Points → BLA 변환 계산
- [ ] 일일 클레임 제한 확인
- [ ] 클레임 후 리셋 확인
- [ ] Top 2 메시지 보상 확인

### 9. Top Messages 테스트
- [ ] 24시간 내 메시지 표시
- [ ] 반응 수 기준 정렬
- [ ] Top 2 메시지 하이라이트

### 10. 데이터베이스 테스트
- [ ] Supabase 연결 확인
- [ ] 테이블 생성 확인
- [ ] Storage 버킷 접근 확인
- [ ] Real-time 업데이트 확인

### 11. 에러 핸들링
- [ ] 네트워크 오류 처리
- [ ] 트랜잭션 실패 처리
- [ ] 환경 변수 누락 처리
- [ ] 잘못된 주소 처리

### 12. UI/UX 테스트
- [ ] 반응형 디자인 확인
- [ ] 로딩 상태 표시
- [ ] 에러 메시지 표시
- [ ] 성공 메시지 표시

## 🚀 메인넷 배포 전 최종 확인

- [ ] 모든 테스트넷 테스트 통과
